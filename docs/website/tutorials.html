1. Installation

Prerequisites

  • Go 1.26+
  • MySQL 8.0+
  • Redis 7.0+
  • Docker (for workflow executor)
  • Task runner (taskfile.dev)

Clone and Build

git clone https://github.com/flowline-io/flowbot.git
cd flowbot

# Copy and edit the config
cp docs/config/config.yaml flowbot.yaml
vim flowbot.yaml

# Install Go tool dependencies
go mod download

# Build the server
go tool task build

Configuration

Edit flowbot.yaml with your environment settings: 

listen: ":6060"

store_config:
  use_adapter: mysql
  adapters:
    mysql:
      dsn: "root:password@tcp(localhost:3306)/flowbot?parseTime=True"

redis:
  addr: "localhost:6379"

platform:
  slack:
    enabled: true
    bot_token: "xoxb-..."
  discord:
    enabled: true
    bot_token: "..."

Database Migration

# Migrations run automatically on startup.
# Or run manually:
go tool task migrate

Run

# Direct
./bin/flowbot

# With live reload (development)
go tool task air

# Docker
docker build -f deployments/Dockerfile -t flowbot .
docker run -p 6060:6060 -v $(pwd)/flowbot.yaml:/opt/app/flowbot.yaml flowbot

Verify

# Check health
curl http://localhost:6060/livez

# View API docs
open http://localhost:6060/swagger/index.html

2. App Discovery & Capability Binding

Scan for Apps

Place your apps in the homelab directory with docker-compose.yaml files. Flowbot scans and registers them automatically. 

# Expected structure:
/home/<user>/homelab/apps/
  archivebox/
    docker-compose.yaml
  miniflux/
    docker-compose.yaml
  kanboard/
    docker-compose.yaml
# Verify discovered apps via API
curl -H "X-AccessToken: sk-..." http://localhost:6060/hub/apps

Bind Apps to Capabilities

Binding tells Flowbot which capability to use for each app: 

# List available capabilities
curl -H "X-AccessToken: sk-..." http://localhost:6060/hub/capabilities

# Bind archivebox to the archive capability (via Hub API or config)

3. Using Capabilities

Once bound, you can invoke capabilities through any interface:

REST API

# List bookmarks
curl -H "X-AccessToken: sk-..." \
  "http://localhost:6060/service/bookmark?limit=20"

# Create a bookmark
curl -X POST \
  -H "X-AccessToken: sk-..." \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com","title":"Example"}' \
  http://localhost:6060/service/bookmark

# Get a specific bookmark
curl -H "X-AccessToken: sk-..." \
  http://localhost:6060/service/bookmark/bkm_a1b2c3

# Delete a bookmark
curl -X DELETE \
  -H "X-AccessToken: sk-..." \
  http://localhost:6060/service/bookmark/bkm_a1b2c3

Chat Bot

# In Discord/Slack, capability commands are available as:
/bookmark list
/bookmark add https://example.com --tags=homelab
/bookmark delete bkm_a1b2c3
/archive submit https://example.com
/reader feeds
/kanban board

CLI

# Using the admin CLI
flowbot-cli capability invoke bookmark list --limit 20
flowbot-cli capability invoke bookmark create \
  --url "https://example.com" --title "Example"
flowbot-cli capability invoke archive submit \
  --url "https://example.com"

4. Creating Declarative Pipelines

Pipelines are defined in YAML and triggered by events, cron, or webhooks.

Define a Pipeline

Create pipelines/archive-on-bookmark.yaml:

# When a new bookmark is saved, archive it and notify
trigger:
  event: "bookmark.created"

steps:
  - action: archive.submit
    input: $.event.url
    idempotency_key: "archive-{{.event.id}}"
    timeout: 30s

  - action: notify.send
    input:
      channel: slack
      message: "Archived: {{.steps.archive-submit.url}}"
    condition: "{{.steps.archive-submit.status}} == 'success'"

Pipelines with Cron Triggers

# Weekly backup pipeline
trigger:
  cron: "0 2 * * 0"    # Every Sunday at 2 AM

steps:
  - action: reader.export
    input:
      format: json
  - action: archive.submit
    input: $.steps.reader-export.output

Pipelines with Webhook Triggers

# Trigger via external webhook
trigger:
  webhook: "github-push"

steps:
  - action: notify.send
    input:
      channel: discord
      message: "Push to {{.event.branch}}: {{.event.commit_message}}"

Pipeline Observability

Every pipeline run is audited. Check execution history:

# Via API (endpoint depends on pipeline module)
flowbot-cli pipeline history archive-on-bookmark
flowbot-cli pipeline replay archive-on-bookmark --run-id=run_xyz

5. Building Workflows

Workflows compose multiple capability steps into a DAG for complex automation.

Example: RSS Digest Workflow

# Daily: fetch feeds, summarize with LLM, save bookmarks, notify
trigger:
  cron: "0 9 * * *"

steps:
  reader.fetch:
    inputs: ["https://example.com/feed.xml"]
  llm.summarize:
    input: $.steps.reader-fetch.items
    prompt: "Summarize these articles in 3 bullet points each"
  bookmark.create:
    input: $.steps.reader-fetch.items
  notify.send:
    input:
      channel: slack
      message: "Daily digest ready: {{len .steps.bookmark-create.items}} articles saved"

Step Types Reference

Type Example Usage
Capability Invoke any capability: bookmark.create, archive.submit, reader.fetch
Message Send a notification through the notify dispatcher
Fetch HTTP GET/POST to external URLs
Feed Parse RSS/Atom feeds from URLs
LLM Generate text, summarize, classify with AI
Docker Run commands in containers
Grep Filter previous step output by pattern
Unique Deduplicate items from previous step
Torrent Manage torrent downloads

6. Configuring Notifications

Flowbot supports 4 notification channels: Slack, Pushover, ntfy, and Message Pusher. 

# flowbot.yaml
notify:
  slack:
    enabled: true
    bot_token: "xoxb-..."
    default_channel: "#alerts"
  pushover:
    enabled: true
    api_token: "..."
    user_key: "..."
  ntfy:
    enabled: true
    server: "https://ntfy.sh"
    topic: "flowbot-alerts"
  message_pusher:
    enabled: true
    api_url: "https://example.com/push"

7. Production Deployment

Docker Compose Stack

version: "3.8"
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: flowbot
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

  flowbot:
    build: .
    ports:
      - "6060:6060"
    volumes:
      - ./flowbot.yaml:/opt/app/flowbot.yaml
    depends_on:
      - mysql
      - redis
    environment:
      - GOMEMLIMIT=256MiB

volumes:
  mysql_data:
  redis_data:

9. Troubleshooting

Common Issues

Symptom Check
Server won't start MySQL and Redis running? flowbot.yaml DSN correct?
Auth errors (401) Include X-AccessToken header. Token valid?
Capability returns 404 App bound to capability? Hub shows the binding?
Pipeline not triggering Redis Stream reachable? Events persisting to data_events table?
Pagination returns empty Cursor expired? Try without cursor for first page.

Logs

# Structured logging via flog (JSON format by default)
# Set log level in flowbot.yaml:
log:
  level: debug    # debug, info, warn, error

Metrics

# Prometheus metrics
curl http://localhost:6060/metrics
